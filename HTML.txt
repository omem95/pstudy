야무의 Front-End Master

1. HTML이란?
HTML의 HT = HyperText(링크형식의 시스템), M = Markup(구조를 설정할 때 사용), L = Language(언어)
>> 구조를 설계할 때 사용되는 언어로 하이퍼링크 시스템 활용

HTML의 확장자는 HTM(L)

HTML은 단순한 텍스트 파일인데 웹브라우저가 해석해서 화면에 렌더링해줌


2.시멘틱 마크업
txt파일과 다르게 html파일은 웹브라우저가 해석을 해서 구조를 설계함

이 구조설계는 웹브라우저에서 우클릭 -> 검사 -> Elements에서 확인 가능

사람이 작성한 컨텐츠를 브라우저가 해석하며 자동으로 설계를 구성하는 것을 마크업이라고 함

HTML은 웹사이트 콘텐츠를 설명하는데 사용되는 마크업 언어로 콘텐츠의 의미 설명이 유일한 목적

HTML엔지니어는 브라우저가 올바르게 해석할 수 있도록 하는 것이 목표

HTML의 요소는 태그로 시작과 끝을 감싸줘야함


3. 기본 문법
웹페이지는 헤드 영역, 바디 영역으로 구성됨

자주 사용되는 용어
elements 요소
open tag 여는 태그
close tag 닫는 태그
attribute 속성
value 값

기본 문법 : <태그이름 속성="값"> 컨텐츠 </태그이름> ->태그 처음부터 끝까지가 요소

브라우저에서 요소에 커서를 올렸을 때 작게 표시되는 창을 툴팁이라고 함, title 속성 값이 출력됨

코드 중 사용자가 보는 것은 body 내부, 브라우저에 정보를 주는 것은 head 내부

html태그는 내부 컨텐츠의 뿌리가 되기 때문에 root elements라고 함

글자가 깨져보인다면 글자 해석(인코딩)에 실패한 것

브라우저의 인코딩 확인 방법 : 우클릭 -> 검사 -> Console -> document.characterSet입력

인코딩 설정 방법 : 헤드태그 내부에 <meta charset="utf-8">


4. 텅 빈 요소
meta태그처럼 닫는 태그가 없는 경우가 있는데, 내용이 없기 때문에 앞뒤로 감쌀 이유가 없기 때문

이를 텅 빈 요소, 비어있는 요소라고 함


5. 표준 호환모드
각 요소의 내부에 속하는 요소는 자식이라고 표현할 수 있으며 들여쓰기로 보기 편하게 표시

html과 head, head와 body, body와 html태그 사이에는 아무것도 없어야함(규칙)

<!DOCTYPE html> 코드의 유무로 표준/비표준 문서 표시, 위치는 반드시 최상단

태그는 대소문자 모두 작성 가능하지만 가급적 소문자 사용 권장

Console에 document.doctype 입력 시 표준/비표준 확인 가능

7. 제목과 단락
h태그는 제목을 뜻하며 레벨을 가짐

h1 태그는 전체에서 한번만 사용, 2단계부터는 여러개 사용 가능

주석은 <!--  -->으로 사용하며 웹브라우저에 출력하지 않을 수 있게됨


8. 이미지와 피규어
웹 문서에서 준로 사용되는 이미지 포멧 : JPG, GIF, PNG(비트맵 그래픽 파일) SVG(벡터 그래픽 파일)

JPG 이미지는 압축률이 높고 다양한 색상을 처리하도록 설계되어 사진이나 복잡한 그래픽 이미지에 많이 사용되나 GIF, PNG와 달리 투명 픽셀 불허

PNG는 사진이나 애니메이션을 제외한 모든 유형에 적합

동일 품질일 경우 JPG보다 PNG가 크기가 크지만 투명처리가 가능해 아이콘, 로고, 다이어그램 등에 사용하기 좋음

GIF는 표현 가능 색상이 256가지로 제한되어 사진에 적합하지 않지만 애니메이션 적용이 가능해 단순 그래픽 애니메이션에 사용하기 좋고 투명처리가 가능하나 PNG보다 표현력이 떨어짐

SVG는 벡터 기반 그래픽 포멧으로 품질 손실 없이 확대, 축소가 가능해 반응형 웹 디자인에 적합

이미지는 img 태그로 불러옴

소스 속성은 src로, alternate text속성은 alt로 나타냄

alt는 이미지 경로가 잘못됐거나 깨졌을 경우 대체되는 텍스트를 출력하는 기능

한글 등의 문서는 내부에 이미지를 포함하지만 HTML문서는 이미지를 포함하지 않고 연결해서 사용함

캡션과 이미지를 묶기 위해 figure태그 사용

figure태그 내부에 이미지 태그와 캡션 내용을 담은 figcaption 태그를 담아서 사용

10. 순차/비순차 목록
순서가 중요한지 아닌지에 따라 비순차/순차 목록으로 구분

비순차 목록은 ul태그(unordered list), 순차 목록은 ol태그(ordered list) 사용

각각의 리스트는 li태그(list)를 사용하며 ul, ol 태그 내부에 들어갈 수 있음


11. 앵커와 하이퍼링크
앵커는 웹 사이트 내부에서 특정 위치로 이동하게 해주며 a 태그에 href 속성 사용

<a href="#id1">내용</a>, <a href="#id2">내용</a>처럼 표기하며 #뒤에 특정 id를 적으면 그 id로 지정된 위치로 이동

하이퍼링크는 지정된 다른 웹 사이트로 이동하게 해주며 방법은 앵커와 동일하되 #없이 url 기입

사진과 텍스트 모두에 하이퍼링크를 적용하려면 img태그를 a태그 내부로 이동

<a href="http://shopping.daum.net/event/detail/31889">
      <img src="이미지파일 이름.jpg" alt="">
      내용

ul과 ol태그는 li태그만 감쌀 수 있고 ul이 ul을 감싸거나 ol이 ol을 감싸는 것도 불가능

target="_blank"속성은 하이퍼링크가 새 탭에서 열림

메일 링크로 연결하려면 href="mailto:메일 주소"로 입력

절대 경로는 url 주소 사용, 상대 경로는 현재 HTML문서를 기준으로 상대적인 위치 설정, 루트 경로는 현재 HTML문서의 최상위 루트 경로에서 대상 위치 설정
